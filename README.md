# 百年

百年是一款以「记忆」为核心的人生记录 App。它不是一个功能齐全的效率工具，而是一件更像「日记本+老照片盒子」的私人物件——克制、温暖、有仪式感，帮用户在琐碎日常里留住值得记住的瞬间，并用「时间线」和「百年」两个尺度重新看见自己的生活。

## 当前版本功能

- 月历首页：查看每一天，已记录日期会被点亮。
- 每日记录：支持标题、正文、心情、标签。
- 图片记录：可从系统相册选择图片并随记录保存。
- 时间线：按时间倒序浏览所有记忆。
- 百年视图：以 100 年为尺度展示人生年份，记录过的年份会被点亮。
- 计划：轻量的 TODO 清单，把想做的事列出来，一条条划掉。
- 我的：设置出生年份/头像，查看应用信息和系统应用设置。

## 设计思想

这一节记录产品与视觉设计上的核心决策，是后续所有 UI 改动应该遵循的「设计宪法」，而不只是一次性的说明。

### 1. 产品气质：情感优先于效率

百年要解决的不是「管理」，而是「留住」和「回望」。因此所有设计决策的第一优先级是**情感浓度**，其次才是效率与信息密度：

- 文案上尽量用有温度的口语化表达，而不是功能性术语。例如首页问候语「下午好」「晚上好」、空状态文案「这里空空如也，点击右下角「+」，记录第一件值得记住的事」，都是在营造「陪伴感」而不是「工具感」。
- 页面标题也遵循这个原则：「回忆」而不是「首页」，「计划」而不是「待办事项」，用词更接近日常说话，而不是后台管理系统的措辞。
- 低频但情感浓度高的页面（如「百年」——审视整个人生尺度）不适合放在触手可及的位置，因为它需要用户带着「主动探访」的心态进入，常驻入口反而会消解这种仪式感。所以「百年」收在「我的」页面里，而不是占用底部导航位。

### 2. 信息架构：三个高频 tab + 次级页面 overlay

底部导航只保留全 App 最高频、最核心的三个入口——**回忆 / 时间线 / 计划**，理由：

- 「回忆」是记录动作的起点（首页），「时间线」是回看的主视角，「计划」是唯一的多任务管理场景，三者都是用户可能每天都要用到的。
- 「日历」「我的」「百年」「计划详情」等更偏「浏览/设置/深度查看」性质的页面，不占用底部导航位，而是以**次级页面（overlay）**的形式浮现在主内容之上，点击返回箭头退出。这样可以保证底部导航栏本身足够克制、不拥挤，同时次级页面依然可以做得足够丰富。
- 判断一个功能该不该放进底部 tab 的标准：是不是「高频到值得用户一步直达，且值得抢占一直可见的屏幕空间」。曾经把「计划」作为「我的」页面里的二级入口，用户反馈「入口太深」，说明这类会被反复使用、需要随时查看/编辑的功能列表，应该直接提升为 tab，而不是藏在设置类页面里。

### 3. 视觉语言：暖白底色 + 墨绿主题色 + 手感圆角

配色克制、低饱和度，参照实体笔记本/相册的质感（`res/values/colors.xml`）：

- 背景 `bg #F7F5F0` 是暖白而不是纯白，减少屏幕的冷感、数字感。
- 主题色 `accent #1B6B5A` 是墨绿色，安静、耐看，不是常见互联网产品爱用的高饱和蓝/红，避免「工具软件」的既视感。
- 文字层级用 `ink`（正文）/`muted`（次要）/`muted_light`（更弱提示）三档灰度过渡，而不是简单的黑白二分。
- 卡片统一用大圆角（`card_elevated`）+ 轻阴影，营造「纸张叠放」的实体质感，而不是扁平方块。

### 4. 图标与底部导航：统一的「手绘感」几何语言

`view/TabIconView.java` 里的图标遵循统一规则，而不是各画各的：

- 统一描边宽度（`s * 0.105`）、统一圆角线帽/线接，让所有图标在粗细、密度上是「同一套字体」。
- 每个图标都承载一点隐喻而不是纯功能符号：「回忆」是圆润心形（情感入口），「时间线」是一条主轴串起三个圆点（回看走过的路），「计划」是一枚打勾的圆角方框（划掉一件事的完成感）。
- 底部导航栏采用「胶囊容器 + 三个等宽 tab」的布局，每个 tab 始终展示「图标 + 文字 + 选中态小圆点」，不做「选中态才展开文字」这类会牺牲可读性的动效——底部导航首先要让用户一眼看清楚自己在哪、能去哪，其次才是动效锦上添花。
- 切换 tab 的动效只做「颜色渐变（灰→墨绿）+ 圆点淡入淡出 + 字重切换」，克制地反馈状态变化，不做过度的位移或形变。

### 5. 交互原则

- 返回路径要符合直觉：次级页面统一「关闭 overlay 回到底部 tab」，不做多层 overlay 嵌套跳转（例如「计划详情」返回时直接回到「计划」tab 列表，而不是先回一个中间页）。
- 涉及内容改动（如换头像、编辑记录）后关闭次级页面时，要顺带刷新一下底层 tab 的展示，保证信息始终一致，不需要用户手动下拉刷新。
- 优先本地优先（local-first）存储用户数据（见 `data/TodoRepository.java`），减少对外部服务的依赖，让核心记录能力始终可用、足够可靠。

> 后续新增页面/组件时，请先对照以上五条判断：这个功能该不该常驻底部？文案是否够有温度？配色/圆角/图标风格是否与现有语言一致？交互路径是否符合「次级页面 overlay」的既有模式？如果某次改动必须打破以上约定，应当在这里同步更新说明，保持这份设计思想与代码演进同步。

## 运行

在项目根目录执行：

```bat
.\gradlew.bat assembleDebug
```

生成的调试包位于：

```text
app\build\outputs\apk\debug\app-debug.apk
```

也可以直接用 Android Studio 打开本目录运行。
